# v2 Architecture Design: UI/API Split with FastAPI Backend

**Date:** 2026-05-06  
**Branch:** `v2/main`  
**Status:** Approved — pending implementation plan

---

## Goal

Split the existing React + Supabase SPA into two independent applications:
- `ui/` — React 18 + Vite (no Supabase SDK)
- `api/` — Python FastAPI (owns all backend logic, DB access, auth, proxies)

All Supabase client SDK usage removed from frontend. Backend uses Supabase Postgres directly via SQLAlchemy ORM + asyncpg. Supabase OAuth kept but owned entirely by FastAPI. Zero functionality changes.

### Schema Strategy

- **Alembic = source of truth** for all schema changes
- Existing DB: run `alembic stamp head` (marks baseline, no migrations applied)
- New changes: write Alembic migration → `alembic upgrade head` → `scripts/dump_schema.sh` → commit both
- `supabase/schema.sql` = auto-generated via `pg_dump --schema-only --no-owner --no-privileges -n public $DATABASE_URL`
- Never hand-edit `schema.sql`

## Section 2: Auth Flow

Backend-owned PKCE OAuth. Frontend has zero Supabase SDK dependency.

### OAuth Dance

```
1. User clicks "Sign in" → window.location = '/api/auth/google'

2. GET /api/auth/google:
   - generate code_verifier + code_challenge (PKCE S256)
   - store code_verifier in temp httpOnly cookie (SameSite=Lax, Secure, 60s TTL)
   - construct Supabase OAuth URL with code_challenge
   - HTTP 302 → Supabase OAuth → Google

3. Google → Supabase → GET /api/auth/callback?code=...

4. GET /api/auth/callback:
   - read code_verifier from temp cookie
   - POST https://{project}.supabase.co/auth/v1/token
       grant_type=pkce, code=..., code_verifier=...
   - receive access_token + refresh_token
   - set httpOnly cookie: access_token  (SameSite=Lax, Secure, 7d TTL)
   - set httpOnly cookie: refresh_token (SameSite=Lax, Secure, 30d TTL)
   - clear verifier cookie
   - HTTP 302 → frontend /

5. Every API request: browser sends cookies automatically
```

### JWT Validation

```python
# dependencies.py
async def get_current_user(request: Request) -> UUID:
    token = request.cookies.get("access_token")
    payload = jwt.decode(
        token,
        SUPABASE_JWT_SECRET,
        algorithms=["HS256"],
        audience="authenticated",   # validates aud claim
    )
    return UUID(payload["sub"])
```

### Token Refresh + Logout

```
POST /api/auth/refresh → exchange refresh_token → set new access_token cookie
POST /api/auth/logout  → clear access_token + refresh_token cookies
GET  /api/auth/me      → current user + profile row
```

### CSRF Protection

- All cookies: `SameSite=Lax`
- All mutating endpoints: FastAPI middleware rejects requests missing `X-Requested-With: XMLHttpRequest`
- CORS: explicit origin list only, never `*` when `credentials: include`

## Section 3: API Endpoints

### Auth — `/api/auth/`
```
GET  /google      → PKCE init, redirect to Supabase OAuth
GET  /callback    → exchange code, set cookies, redirect to /
POST /logout      → clear cookies
POST /refresh     → exchange refresh_token → new access_token cookie
GET  /me          → current user + profile
```

### Sessions — `/api/sessions/`

Note: `{code}` = invite_code (alphanumeric string e.g. "AB3X7K"). `{id}` = session UUID. Frontend fetches session by code first, then uses `id` for all subsequent mutations.

```
POST   /                      → create session (caller = host)
GET    /{code}                 → get session by invite code → returns session with id
POST   /{id}/join              → join session
DELETE /{id}/leave             → leave session
PATCH  /{id}/end               → end session (host only)
PATCH  /{id}/repeat-mode       → set repeat mode (DJ only)
POST   /{id}/dj                → pass DJ token (host only)
POST   /{id}/heartbeat         → presence heartbeat (every 30s)
GET    /{id}/participants       → list participants
GET    /{id}/queue             → get full queue (ordered by position)
POST   /{id}/queue             → add item (calls Odesli, stores full meta)
POST   /{id}/queue/next        → play next (DJ/host only)
POST   /{id}/queue/skip        → force skip (DJ/host only)
GET    /{id}/stream            → SSE stream (event_bus keyed by session UUID)
```

### Queue Items — `/api/items/`
```
PUT    /{item_id}/youtube-link → patch resolved YouTube URL
POST   /{item_id}/votes        → cast skip vote
DELETE /{item_id}/votes        → remove skip vote
```

### Profiles — `/api/profiles/`
```
GET   /{user_id}   → get profile
PATCH /me          → update preferred_platform
```

### Songs — `/api/song/`
```
GET /?url=...      → Odesli lookup (returns title, artist, thumbnailUrl, platformLinks)
```

### YouTube — `/api/youtube/`
```
GET /?q=...        → SearXNG search → YouTube video ID + title
```

### Flags — `/api/flags/`
```
GET /              → all feature flags { key, enabled }[]
```

### Odesli Behavior (unchanged from current)

- Add-to-queue: calls Odesli → stores title + artist + thumbnailUrl + platformLinks immediately
- No deferred resolution — full metadata stored on queue insert
- `song_service.resolve_song_meta(url)` → single Odesli call, returns complete meta

## Section 4: SSE + Realtime

### Architecture

```
DB change
  → Supabase Realtime (backend subscription)
  → FastAPI event listener (supabase-py)
  → event_bus.publish(session_id, event)
  → asyncio.Queue per SSE client
  → SSE response stream → browser
```

FastAPI holds one Supabase Realtime channel per active session (opened on first SSE subscriber, closed when last subscriber disconnects).

### event_bus.py

```python
_rooms: dict[str, list[asyncio.Queue]] = {}

async def subscribe(session_id: str) -> asyncio.Queue
async def unsubscribe(session_id: str, q: asyncio.Queue)
async def publish(session_id: str, event: dict)
# publish also closes Supabase channel if no subscribers remain
```

### Supabase Realtime → Event Mapping

| Table | Event type | Payload |
|---|---|---|
| `sessions` | `session_updated` | full session row |
| `queue_items` | `queue_changed` | refetch all items for session |
| `session_participants` | `participants_changed` | refetch all participants |
| `skip_votes` | `votes_changed` | `{ queue_item_id, count, user_ids[] }` |

### SSE Endpoint

```python
# GET /api/sessions/{id}/stream
# - validates JWT
# - looks up session by UUID → subscribes to event_bus keyed by session UUID
# - opens Supabase Realtime channel if not already open for this session
# - yields: data: {"type": ..., "payload": ...}\n\n
# - on disconnect: unsubscribe, close Supabase channel if no subscribers remain
```

### Frontend SSE (sse.js)

```javascript
export function openSSE(sessionCode, handlers) {
  const es = new EventSource(`/api/sessions/${sessionCode}/stream`, { withCredentials: true })
  es.onmessage = e => {
    const { type, payload } = JSON.parse(e.data)
    handlers[type]?.(payload)
  }
  es.onerror = () => handlers.onReconnect?.()  // triggers REST refetch
  return () => es.close()
}
```

### Reconnect Strategy

On `onerror`: hooks perform REST refetch (same pattern as current Supabase hooks), then SSE auto-reconnects via browser. No `Last-Event-ID` buffer needed — hooks already refetch full state on every realtime event.

### repeat_mode=song Handling

Frontend owns this entirely. `YouTubeAutoPlayer` reads `repeat_mode` from session state (delivered via `session_updated` SSE event). If `song`, seeks to 0 and replays locally — never calls `play_next`. Identical to current behavior.

## Section 5: SOLID Structure

### S — Single Responsibility
- Router: HTTP parsing + response serialization only
- Service: business logic + orchestration, no HTTP/DB awareness
- Repository: DB queries only, returns ORM models
- Model: SQLAlchemy mapped class, data shape only
- Schema: Pydantic wire format only

### O — Open/Closed

```python
# repositories/base.py
class AbstractRepository(ABC):
    @abstractmethod
    async def get_by_id(self, id: UUID) -> Optional[Any]: ...
    @abstractmethod
    async def create(self, **kwargs) -> Any: ...

# repositories/session_repo.py — extends, never modifies base
class SessionRepository(AbstractRepository):
    async def get_by_code(self, code: str) -> Optional[Session]: ...
```

### L — Liskov
All concrete repos fully implement `AbstractRepository`. No `NotImplementedError` stubs in production paths.

### I — Interface Segregation
Each router imports only the service it needs. No god services. `SessionService` has no knowledge of `QueueService`.

### D — Dependency Inversion

```python
# dependencies.py — all wiring here
async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with AsyncSessionLocal() as session:
        yield session

def get_session_repo(db: AsyncSession = Depends(get_db)) -> SessionRepository:
    return SessionRepository(db)

def get_session_service(repo: SessionRepository = Depends(get_session_repo)) -> SessionService:
    return SessionService(repo)

# routers/sessions.py
@router.post("/")
async def create_session(svc: SessionService = Depends(get_session_service)):
    ...
```

### ORM Models (SQLAlchemy async)

```python
# models/session.py
class Session(Base):
    __tablename__ = "sessions"
    id: Mapped[UUID] = mapped_column(primary_key=True, default=uuid4)
    invite_code: Mapped[str] = mapped_column(unique=True, nullable=False)
    host_user_id: Mapped[Optional[UUID]] = mapped_column(ForeignKey("profiles.id"))
    dj_user_id: Mapped[Optional[UUID]] = mapped_column(ForeignKey("profiles.id"))
    status: Mapped[str] = mapped_column(default="active")
    repeat_mode: Mapped[str] = mapped_column(default="none")
    max_participants: Mapped[int] = mapped_column(default=20)
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    ended_at: Mapped[Optional[datetime]]
    expires_at: Mapped[datetime]
```

### Postgres RPCs (called via SQLAlchemy raw SQL)

```python
# queue_service.py
await db.execute(text("SELECT play_next(:sid, :status, :auth)"), {
    "sid": session_id, "status": skip_status, "auth": check_auth
})
```

`play_next`, `cast_skip_vote`, `pass_dj_token`, `set_repeat_mode` all remain as Postgres functions — called via `text()` through the ORM session. No logic duplication in Python.

## Section 6: Frontend Changes

### Removed
- `@supabase/supabase-js` package
- `src/lib/supabase.js`
- All `supabase.from(...)`, `supabase.channel(...)`, `supabase.auth.*` calls

### New: src/lib/api.js

```javascript
export async function api(path, options = {}) {
  const res = await fetch(`/api${path}`, {
    credentials: 'include',
    headers: {
      'Content-Type': 'application/json',
      'X-Requested-With': 'XMLHttpRequest',
      ...options.headers,
    },
    ...options,
  })
  if (res.status === 401) {
    await fetch('/api/auth/refresh', { method: 'POST', credentials: 'include' })
    return fetch(`/api${path}`, { ...options, credentials: 'include',
      headers: { 'X-Requested-With': 'XMLHttpRequest', ...options.headers }
    })
  }
  return res
}
```
